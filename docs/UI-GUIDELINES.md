> 版本：v0.5（现有架构优化 / Light-first + Themeable）
> 日期：2026-07-16
> 对应 PRD/SPEC：`docs/PRD.md`、`docs/SPEC.md`
> 原型：`docs/prototypes/gravuresse-vnext-prototype.html`
> 截图：`docs/prototypes/screens/`

## 1. 体验定位

### 1.1 主表面

v0.5 默认贴合当前 Gravuresse 真实界面，不另起平台壳：

```text
TitleBar
└─ WorkspaceShell
   ├─ ModuleSidebar：生成 / 视频 / 连接 / 设置
   └─ WorkspaceMain
      ├─ ChatPanel：创作记录、输入、生成设置、参考图
      └─ CanvasPanel：素材区、网格/自由视图、素材详情
```

核心原则：

```text
当前创作记录承载上下文；素材区承载结果；素材详情承载继续创作和来源解释；设置承载偏好和连接。
```

### 1.2 v0.5 用户闭环

```text
输入想法 → 生成图像 → 选择满意结果 → 继续修改 / 转视频 → 整理素材 → 下次打开继续
```

界面必须优先证明这个闭环，而不是展示未来大型工作台。

### 1.3 这版优化什么

| 方向 | 目标 |
|---|---|
| 创作流程 | 减少“下一步在哪”的犹豫 |
| 界面布局 | 保持现有双区架构，信息更有层级 |
| 组件体系 | 同类控件一致，方便后续维护 |
| 主题 | 浅色默认，深色完整可用，支持跟随系统 |
| 语言 | 中文默认，英文可切换，主路径进入 i18n |
| 可维护性 | 新功能能挂在清楚组件和 token 上 |

## 3. 全局布局

### 3.1 Desktop Shell

默认桌面布局：

```text
┌──────────────────────── Title Bar 44px ────────────────────────┐
├─ Module 56 ─┬─ Chat Panel 360–420 ─┬─ Canvas Panel flex ────────┤
│             │                      │  Toolbar / Grid / Detail   │
└─────────────┴──────────────────────┴────────────────────────────┘
```

| 区域 | 默认 | 行为 |
|---|---:|---|
| 标题栏 | 44px | 显示 Gravuresse、版本、自动保存、设置入口 |
| 模块栏 | 56px | 生成/视频/连接/设置，不堆二级导航 |
| 聊天区 | 360–420px | 当前创作记录、输入、生成设置、参考摘要 |
| 素材区 | flex | 网格/自由视图、导入、筛选、批量动作 |
| 素材详情 | 300–340px | 有选中素材时展示，可在窄屏折叠 |
| 设置弹窗 | 860px 左右 | Provider 设置 + 通用偏好 |

窗口较窄时：

1. 优先保留聊天区和素材区；
2. 素材详情可变为抽屉；
3. 会话列表可用弹出层；
4. 工具栏允许横向滚动，但核心动作不可完全隐藏。

### 3.2 主工作区

主工作区分为：

1. **聊天区**：用户输入想法、修改要求、查看任务反馈；
2. **素材区**：展示图像、视频、参考素材；
3. **输入栏**：自然语言输入 + 当前参数 + 当前参考摘要；
4. **素材详情**：生成说明 + 继续动作；
5. **任务状态**：生成成功、失败、恢复状态都写回当前记录。

## 4. 核心屏幕规范

### 4.1 当前创作记录工作区

截图：`01-current-workspace-light.png`

必须包含：

- 当前应用标题和版本；
- 模块栏；
- 当前创作记录入口；
- 聊天上下文；
- 输入栏和生成设置摘要；
- 素材网格；
- 选中素材详情；
- 自动保存状态；
- “作为参考继续生成 / 生成视频”入口。

禁止：

- 把首页做成营销页；
- 要求用户先选择复杂模式；
- 把画布连线当生成逻辑；
- 默认界面堆内部工程词。

### 4.2 素材详情与继续创作

截图：`02-asset-detail-continue.png`

素材详情是 P0 的透明度入口，默认显示：

- 预览；
- prompt / negative prompt；
- 模型 / 连接；
- 比例 / 分辨率 / 时长；
- 父素材；
- 参考素材；
- 远程 taskId；
- 创建时间 / 错误信息。

主操作：

- **作为参考继续生成**；
- **生成视频**；
- **加入当前输入**；
- **保存到本地**。

### 4.3 视频任务恢复

截图：`03-video-task-recovery.png`

要求：

- queued/running 任务可见；
- 能恢复轮询时显示“运行中”；
- 连接缺失或远程状态不明时显示“需手动检查”；
- 不伪造成成功或失败；
- 重试创建新任务，不覆盖旧错误和旧 taskId。

### 4.4 设置 / 主题 / 语言

截图：`04-preferences-theme-language.png`

设置页应保持一个体系：

```text
左侧：Provider 设置
右侧：账号 / API 密钥 / 自定义中转 / 默认模型搭配
顶部或折叠区：通用设置（语言、主题、字号、功能开关）
```

要求：

- 通用设置可发现，不藏太深；
- 语言、主题、字号即时保存；
- Provider 变更继续即时保存；
- 密钥不回显；
- 深色和英文不是单独页面，而是当前设置里的偏好。

### 4.5 深色主题预览

截图：`05-dark-theme-preview.png`

要求：

- 和浅色同一布局；
- 不改变信息架构；
- 使用同一组件和 token；
- 暗色下边界、输入框、卡片、disabled 状态清楚；
- 媒体内容仍然是视觉主角。

### 4.6 UI 规范板

截图：`06-ui-specification.png`

必须包含：

- 当前布局；
- Light/Dark token；
- 字体层级；
- 组件样例；
- 主题和语言规则；
- Do / Don’t；
- 可维护性门禁。

## 5. 视觉规范

### 5.1 风格

默认风格：

- 浅色、克制、本地桌面工具感；
- 少装饰，细边界，清楚层级；
- 媒体内容承担视觉张力；
- 状态不能只靠颜色，需要文字标签；
- 深色主题可以更酷，但不改变产品架构。

### 5.2 语义色彩 Token

使用现有 `global.css` token 体系。

| Token | 用途 |
|---|---|
| `--bg-workspace` | 应用工作区背景 |
| `--bg-primary` | 主内容背景 |
| `--bg-elevated` | 弹窗 / 抬升面 |
| `--bg-surface` | 卡片 / 输入 / 次级面 |
| `--bg-hover` | hover |
| `--text-primary` | 主文本 |
| `--text-secondary` | 次级文本 |
| `--text-muted` | 辅助文本 |
| `--border-subtle` | 弱边界 |
| `--border-default` | 默认边界 |
| `--accent` | 主操作 / 选中 |
| `--accent-soft` | 主操作弱背景 |
| `--success` | 成功 / 可恢复 |
| `--danger` | 失败 / 需检查 |

不要在新组件里硬编码主题色。

### 5.3 字体

```css
font-family: "Segoe UI Variable", "Microsoft YaHei UI", "PingFang SC", system-ui, sans-serif;
```

| 层级 | 尺寸 | 用途 |
|---|---:|---|
| 页面标题 | 17–20px | 当前创作记录 / 设置页标题 |
| 面板标题 | 13–15px | 素材详情、任务队列、设置分组 |
| 正文 | 13–14px | 消息、详情正文 |
| 辅助 | 11–12px | 元信息、状态、时间 |
| 代码/ID | 10–11px | taskId、文件名、字段名 |

字号设置：

| 设置 | 用途 |
|---|---|
| small | 高密度桌面使用 |
| medium | 默认 |
| large | 可读性优先 |

## 6. 组件规范

### 6.1 Button

| 类型 | 用途 |
|---|---|
| Primary | 发送、继续生成、确认动作 |
| Secondary | 导入、导出、刷新 |
| Ghost | 工具栏、弱动作 |
| Danger | 删除、清空 |
| Icon | 标题栏、工具栏 |

规则：

- 同一类型在 light/dark 下只换 token；
- disabled 状态必须明显；
- 按钮文案使用 i18n key。

### 6.2 Pill / Status

用途：

- 已保存；
- 运行中；
- 完成；
- 需手动检查；
- 图像 / 视频 / 参考图；
- 比例 / 分辨率 / 模型能力。

状态必须有文字，不只靠颜色。

### 6.3 AssetCard

必须显示：

- 媒体缩略图；
- 作品名；
- 类型 / 模型 / 关键参数；
- 选中态；
- 参考图 / 视频 / 素材等轻标签。

可提供 hover 动作，但主动作仍应在 AssetDetail 中稳定可见。

### 6.4 AssetDetail

默认分组：

1. 预览；
2. 生成说明；
3. 参考素材；
4. 继续创作动作；
5. 错误或任务状态。

### 6.5 TaskCard

| 状态 | 文案 | 颜色 |
|---|---|---|
| 运行中 | 运行中 xx% | accent |
| 完成 | 完成 | success |
| 需手动检查 | 需手动检查 | danger |
| 失败 | 失败，可重试 | danger |
| 未知 | 状态未知 | muted / danger soft |

未知状态必须说清楚，不自动当成失败。

### 6.6 PreferenceRow

用于：

- 语言；
- 主题；
- 字号；
- 参考图开关；
- 视频工作区开关；
- 自动保存。

规则：

- label + control + hint 三段清楚；
- 保存中和保存失败可见；
- 切换后即时生效。

## 7. Do / Don’t

### Do

- 保留当前简洁架构；
- 默认浅色，深色作为主题；
- 中文默认，英文可切换；
- 素材详情解释作品怎么来的；
- 从素材直接继续生成或转视频；
- 任务状态未知时明确提示用户；
- 连接页按任务展示能力；
- 新组件使用 token 和 i18n；
- 小步抽组件，小步验证。

### Don’t

- 不新增一整套平台结构；
- 不把画布当执行引擎；
- 不伪造生成进度或恢复结果；
- 不在默认界面堆内部工程词；
- 不替用户预设未经要求的服务；
- 不把深色界面做成另一套产品；
- 不让英文文案散落在组件里；
- 不为了未来扩展牺牲当前简单架构。

## 8. 验收门禁

- [ ] 3 秒内能看出当前创作记录、输入区、素材区和详情区；
- [ ] 选中素材后能看到“作为参考继续生成”和“生成视频”；
- [ ] 素材详情能解释 prompt、模型、父素材和参考素材；
- [ ] 视频任务恢复状态不会伪造成成功/失败；
- [ ] 设置中能找到语言、主题、字号；
- [ ] 浅色和深色布局一致；
- [ ] 中文和英文主路径文案完整；
- [ ] 连接页不回显秘密；
- [ ] 默认界面不出现复杂内部概念；
- [ ] 新组件不硬编码主题色；
- [ ] UI 规范不引导新增大架构。
