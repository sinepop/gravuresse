> 版本：v0.4（现有架构优先 / 当前闭环）
> 日期：2026-07-16
> 对应 PRD/SPEC：`docs/PRD.md`、`docs/SPEC.md`
> 原型：`docs/prototypes/gravuresse-vnext-prototype.html`

## 1. 体验定位

### 1.1 主表面

v0.4 不再把 Gravuresse 表达成一个新平台。默认产品表面贴近当前项目形态：

- **创作记录**：当前会话，承载聊天、任务和素材；
- **聊天 / 输入区**：表达想法、发起生成、继续修改；
- **素材区**：图像、视频和参考素材；
- **素材详情**：解释作品怎么来的，并提供继续创作动作；
- **任务队列**：显示视频任务状态和重启恢复结果；
- **连接页**：沿用现有连接体系，按任务展示能不能对话、生成图、生成视频。

核心原则：

```text
当前会话就是工作空间；素材详情就是最小来源记录；画布只是整理界面。
```

### 1.2 v0.4 用户闭环

```text
输入想法 → 生成图像 → 保存/整理结果 → 选中结果继续修改或转视频 → 下次打开还能接着做
```

界面必须优先证明这个闭环，而不是展示未来大型工作台。

### 1.3 默认概念边界

默认用户界面允许出现：

| 用户概念 | 说明 |
|---|---|
| 创作记录 | 当前会话 |
| 作品 / 素材 | 图像、视频、参考图 |
| 生成说明 | prompt、模型、尺寸、参考图、父素材 |
| 作为参考继续生成 | 基于选中素材继续做图 |
| 生成视频 | 从选中图像发起视频任务 |
| 任务状态 | 运行中、完成、失败、需手动检查 |
| 连接 | 已配置的模型入口 |

默认界面不出现复杂内部架构词；这些词只允许在 PRD/SPEC 的边界说明中出现，不进入产品默认路径。

## 3. 全局布局

### 3.1 Desktop Shell

```text
┌──────────── Title Bar / Current Record / Status ─────────────┐
├─ Rail ─┬─ Creation Records ─┬─ Chat + Assets ─┬ Asset Detail ┤
│ 56px   │ 288px             │ flexible        │ 332px       │
└────────┴───────────────────┴─────────────────┴──────────────┘
```

| 区域 | 默认 | 行为 |
|---|---:|---|
| 标题栏 | 40px | 显示当前创作记录、自动保存、本地优先状态 |
| 图标栏 | 56px | 全局入口，不堆二级导航 |
| 创作记录栏 | 288px | 当前会话列表 + 进行中任务摘要 |
| 主工作区 | flex | 聊天和素材区并列，是默认操作中心 |
| 素材详情 | 332px | 选中素材后显示生成说明和继续创作入口 |
| 底部状态条 | 30px | 显示 v0.4 门禁和状态 |

窗口较窄时可收起创作记录栏或素材详情，但主工作区必须优先保留。

### 3.2 主工作区

主工作区分为：

1. **聊天区**：用户输入想法、修改要求、任务反馈；
2. **素材区**：展示图像、视频、参考素材；
3. **输入栏**：自然语言输入 + 当前参考摘要；
4. **任务反馈**：生成成功、失败、恢复状态都写回当前记录。

## 4. 核心屏幕规范

### 4.1 当前创作记录工作区

截图：`01-current-workspace.png`

必须包含：

- 当前创作记录标题；
- 会话列表；
- 聊天上下文；
- 素材网格；
- 选中素材详情；
- 自动保存状态；
- “作为参考继续生成 / 生成视频”入口。

禁止：

- 把首页做成营销页；
- 要求用户先选择模式；
- 把画布连线当生成逻辑；
- 在默认界面堆复杂架构词。

### 4.2 素材详情与继续创作

截图：`02-asset-detail-continue.png`

素材详情是 P0 的透明度入口，默认显示：

- prompt / negative prompt；
- 模型 / 连接；
- 比例 / 分辨率 / 时长；
- 父素材；
- 参考素材；
- 远程 taskId；
- 创建时间 / 错误信息。

主操作：

- **作为参考继续生成**；
- **生成视频**；
- 加入当前输入；
- 保存到磁盘。

### 4.3 视频任务恢复

截图：`03-video-task-recovery.png`

要求：

- queued/running 任务可见；
- 能恢复轮询时显示“运行中”；
- 连接缺失或远程状态不明时显示“需手动检查”；
- 不伪造成成功或失败；
- 重试创建新任务，不覆盖旧错误和旧 taskId。

### 4.4 连接与模型

截图：`04-connections-models.png`

连接页回答三个问题：

1. 当前能不能对话；
2. 当前能不能生成图；
3. 当前能不能生成视频。

规则：

- 沿用现有 Provider connections；
- 不新增未经用户要求的预设服务；
- 密钥、token、cookie 不回显；
- 技术详情折叠；
- 连接失败要和任务恢复状态联动。

### 4.5 UI 规范板

截图：`05-ui-specification.png`

必须包含：

- 色彩 token；
- 字体层级；
- 桌面布局；
- 组件样例；
- Do / Don’t；
- v0.4 门禁。

## 5. 视觉规范

### 5.1 风格

- 深色媒体工具，不做营销 landing；
- 低饱和背景 + 暖橙主操作；
- 细边界、少渐变、少装饰；
- 媒体内容承担视觉张力；
- 状态不能只靠颜色，需要文字标签。

### 5.2 色彩 Token

| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#0B0D10` | 应用背景 |
| `--panel` | `#171B20` | 面板背景 |
| `--panel2` | `#1E232A` | 浮层 / 卡片 |
| `--line` | `#2A3038` | 默认边界 |
| `--text` | `#F4F5F7` | 主文本 |
| `--sub` | `#B6BDC7` | 次级文本 |
| `--muted` | `#7F8894` | 辅助文本 |
| `--amber` | `#F0A44B` | 主操作 / 选中 |
| `--green` | `#45B97C` | 成功 / 可恢复 |
| `--red` | `#E96B68` | 错误 / 需检查 |

### 5.3 字体

```css
font-family: "Segoe UI Variable", "Microsoft YaHei UI", "PingFang SC", system-ui, sans-serif;
```

| 层级 | 尺寸 | 用途 |
|---|---:|---|
| 页面标题 | 17–18px | 当前创作记录 / 设置页标题 |
| 面板标题 | 13–15px | 创作记录、素材详情、任务队列 |
| 正文 | 13px | 消息、详情正文 |
| 辅助 | 11px | 元信息、状态、时间 |
| 代码/ID | 10–11px | taskId、文件名、字段名 |

## 6. 组件规范

### 6.1 创作记录 Item

必须显示：

- 标题；
- 消息数；
- 素材数；
- 最近更新时间。

选中态：左侧暖橙线 + 较亮面板背景。

### 6.2 素材卡片

必须显示：

- 媒体缩略图；
- 作品名；
- 类型 / 模型 / 关键参数；
- 选中态；
- 参考图 / 视频 / 素材等轻标签。

### 6.3 素材详情

默认折叠成几组：

1. 预览；
2. 生成说明；
3. 参考素材；
4. 继续创作动作；
5. 错误或任务状态。

### 6.4 视频任务卡

状态：

| 状态 | 文案 | 颜色 |
|---|---|---|
| 运行中 | 运行中 xx% | 暖橙 |
| 完成 | 完成 | 绿色 |
| 需手动检查 | 需手动检查 | 红色 |
| 失败 | 失败，可重试 | 红色 |

未知状态必须说清楚，不自动当成失败。

### 6.5 连接卡

显示：

- 连接名；
- 最近验证；
- 默认模型；
- 可对话 / 可生成图 / 可生成视频；
- 是否需要源图片；
- 失败原因。

不显示：密钥、token、cookie、完整请求头。

## 7. Do / Don’t

### Do

- 当前会话就是默认工作空间；
- 素材详情解释作品怎么来的；
- 从素材直接继续生成或转视频；
- 任务状态未知时明确提示用户；
- 连接页按任务展示能力；
- UI 贴合当前代码结构，少概念、少迁移。

### Don’t

- 不新增一整套平台结构；
- 不把画布当执行引擎；
- 不伪造生成进度或恢复结果；
- 不在默认界面堆内部架构词；
- 不替用户预设未经要求的服务；
- 不为了未来扩展牺牲当前简单架构。

---

## 8. 验收门禁

- [ ] 3 秒内能看出当前创作记录、素材区和详情区；
- [ ] 选中素材后能看到“作为参考继续生成”和“生成视频”；
- [ ] 素材详情能解释 prompt、模型、父素材和参考素材；
- [ ] 视频任务恢复状态不会伪造成成功/失败；
- [ ] 连接页不回显秘密；
- [ ] 默认界面不出现复杂架构词；
- [ ] UI 规范不引导新增大架构。
