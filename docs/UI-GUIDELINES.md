# Gravuresse vNext UI/UX 规范

> 版本：v0.1
> 日期：2026-07-16
> 规范源：根目录 `DESIGN.md`
> 原型：`docs/prototypes/gravuresse-vnext-prototype.html`

---

## 1. 体验定位

### 1.1 主表面

Gravuresse 的项目工作区是 **Operate + Command/Inspect** 表面：

- Operate：用户持续输入、执行、选择、修改和排队；
- Command/Inspect：用户检查素材、配方、上下文、谱系和 Agent 运行；
- Canvas 是辅助 Explore 表面，但不能让浏览感压过任务控制。

因此不使用营销页式大标题、功能卡片和装饰性留白。核心内容在 1280×720 内保持可操作。

### 1.2 设计目标

1. 让画面成为视觉中心，界面退后；
2. 让复杂性按需展开，而不是隐藏不可控；
3. 让“改变什么、保留什么、为什么”一眼可见；
4. 让用户随时知道当前会话、上下文、执行者、模型和任务状态；
5. 让 Pipeline 和无限画布看起来是同一产品的两种视图。

### 1.3 不做什么

- 不复制 Lovart、Runway、Miro 的具体品牌界面；
- 不使用紫蓝 AI 渐变作为品牌；
- 不大面积玻璃模糊；
- 不把每个功能做成同权卡片；
- 不用装饰性 Agent 头像替代执行状态；
- 不用“100%复现”“完整上下文”等虚假文案。

## 2. 信息架构与窗口布局

### 2.1 全局结构

```text
┌──────────── Title Bar / Project Switcher / Global Actions ────────────┐
├─ Project Rail ─┬─ Conversation/Agent ─┬─ Main Work Surface ─┬ Inspector ┤
│ 56 / 240px     │ 320–380px           │ flexible           │ 320–360px │
└────────────────┴──────────────────────┴─────────────────────┴───────────┘
```

### 2.2 尺寸建议

| 区域 | 默认 | 最小 | 行为 |
|---|---:|---:|---|
| 标题栏 | 40px | 40px | Electron drag region，交互控件禁用 drag |
| 图标项目栏 | 56px | 56px | 可展开为 240px 项目列表 |
| 会话/Agent栏 | 344px | 296px | 可收起；用户拖动调整 |
| 主工作面 | 自适应 | 520px | 优先保留媒体和 Pipeline |
| 检查器 | 336px | 300px | 无选中对象时显示项目/任务摘要 |
| 底部任务条 | 32px | 32px | 有任务时出现，可展开队列 |

窗口宽度 < 1180px 时：会话栏和检查器只允许一个展开；< 900px 进入专注模式，通过顶部按钮切换面板。

## 3. 核心屏幕

### 3.1 项目首页 / 新建项目

**目的**：在 30 秒内开始，不把 Provider 配置放在第一步。

结构：
- 最近项目列表；
- “新建项目”主操作；
- 新建弹层选择：Pipeline / 无限画布；
- 名称、用途、第一段想法为可选；
- 若缺少所需连接，在第一次生成时再提示设置。

模式卡必须说明差异：
- Pipeline：步骤清晰、快速收敛、适合第一次创作；
- 无限画布：自由分支、复杂关系、适合已有素材或专业探索。

### 3.2 Pipeline 工作区

**主任务**：把模糊想法推进到可交付媒体。

主工作面横向/纵向展示 5 个阶段：
1. 想法；
2. 创意简报；
3. 图像探索；
4. 选定与修改；
5. 视频（可选）。

每个阶段只能有一个主状态：Idle / Ready / Running / Needs review / Complete / Error。候选媒体在步骤内部形成不等宽 strip，选定项使用琥珀边框和“已选”标签。

主工作面右上角提供“展开为画布”，不是新建另一份项目。

### 3.3 无限画布工作区

**主任务**：自由组织素材、分支和生成关系。

画布内容：
- Brief/文本节点；
- 图片和视频节点；
- Agent/任务节点只在开启“执行轨迹”时显示；
- 谱系线默认仅显示选中节点的一跳关系；
- 空白处右键或 `/` 打开命令菜单；
- 选中媒体后，右侧检查器显示修改、配方与谱系。

工具条只放高频操作：选择、移动、连接、文本、上传、生成。绘图工具收进二级菜单，避免与主要创作任务竞争。

### 3.4 上下文与会话

左侧会话栏必须同时表达：
- 当前会话；
- Fork 来源；
- 哪些内容已 Pin 到项目；
- 本轮将使用哪些上下文；
- 上下文超限时排除了什么。

“新会话”创建空分支；“Fork”继承当前快照。二者文案和图标不可混淆。

### 3.5 连接设置

采用 **Configure** 表面：
- 顶部按能力筛选 Chat / Image / Video；
- 每张连接行显示名称、endpoint host、认证状态、已验证能力、模型数量、最近验证时间；
- “能列模型”和“能真实推理”分两级状态；
- 高级请求模板默认折叠；
- 密钥输入保存后只显示脱敏状态；
- 不显示未经验证的能力为可用。

## 5. 核心组件

### 5.1 ConversationItem

内容：标题、更新时间、分支标记、未完成任务数、Pinned 数量。
状态：default / hover / active / running / error。
禁用：不要用多彩头像表示会话。

### 5.2 ContextManifestPreview

发送按钮上方显示一行摘要：

```text
本轮上下文：Brief v3 · 项目记忆 4 · 当前会话 12 条 · 参考图 2
```

点击展开显示来源、版本、Included/Excluded 和排除理由。高风险的自动摘要修订显示“待确认”。

### 5.3 AgentPlan

运行前显示：
- 步骤名称；
- 串行/并行关系；
- 使用连接；
- 最大调用次数；
- 预计成本（有数据才显示）；
- 运行/编辑/取消。

运行中使用细状态线而非聊天气泡：Queued → Running → Complete/Error。用户随时可停止。

### 5.4 PipelineStep

结构：序号、名称、状态、输入摘要、主产物、候选区、下一步动作。
一个步骤只允许一个高强调 CTA。Running 状态使用低幅进度运动，禁止永久旋转的装饰动画。

### 5.5 AssetCard

媒体优先，信息在 hover/选中时出现。固定内容：
- 缩略图；
- 类型/时长；
- 状态；
- 选定标记；
- 谱系分支数量。

Prompt、Provider 和参数不叠在图上，进入检查器查看。

### 5.6 TransparencySwitch

三段：黑盒 / 灰盒 / 白盒。切换只改变暴露的控制，不改变底层 Recipe。

- 黑盒：变更描述、保持/改变摘要；
- 灰盒：语义分组、变化强度、锁定；
- 白盒：Prompt diff、模型、seed、参数、引用。

### 5.7 StabilityLocks

锁定维度：主体、身份、构图、镜头、环境、风格、光线、色彩。
状态：Off / Prefer / Strict。Strict 若当前模型无法保证，显示琥珀警告，不假装已执行。

### 5.8 ReproducibilityBadge

只使用 High / Medium / Low，不显示伪精确百分比。点击解释影响因素：
- seed 是否被接受；
- 模型版本是否固定；
- 参考素材 hash 是否完整；
- Provider 是否返回执行证据。

### 5.9 RecipeDiff

两列不适合窄检查器，默认使用逐段 diff：
- 保持：中性；
- 新增：绿色文本/图标；
- 删除：红色删除线；
- 改变：琥珀强调。

颜色不能是唯一线索，必须同时有文字和符号。

### 5.10 ConnectionRow

显示：连接名、host、能力标签、状态、模型数、延迟、更多菜单。
认证字段从不在列表中回显。失败状态给出动作：“重新验证”“编辑 endpoint”“查看安全错误”。

## 6. 视觉系统

规范 token 以根目录 `DESIGN.md` 为准。

### 6.1 色彩原则

- 近黑背景：让媒体成为主要色彩来源；
- 琥珀 `#F0A44B`：只用于选中、主操作、焦点和需要用户确认的状态；
- 蓝色仅用于链接/信息；
- 绿色只用于成功，不代表“AI”；
- 红色只用于失败和破坏性操作；
- 不用紫蓝渐变、霓虹边缘或彩色玻璃卡片。

### 6.2 字体

Windows Electron 优先：

```css
font-family: "Segoe UI Variable", "Microsoft YaHei UI", "PingFang SC", sans-serif;
```

技术值使用：

```css
font-family: "Cascadia Mono", "SFMono-Regular", Consolas, monospace;
```

层级：
- 窗口/项目标题：18–24px / 600；
- 面板标题：14px / 600；
- 正文与控件：13–14px / 400–500；
- 元数据：11–12px；
- 不使用超大展示字体。

### 6.3 空间与形状

- 4px 基础网格，主要节奏 8/12/16/24；
- 控件 radius 6px；卡片 8px；浮层 12px；
- 媒体卡 radius 6px，避免过度圆润；
- 小控件高度 28px，标准控件 36px，主输入 44px；
- 图标按钮可点击区域至少 36×36px；高频触摸场景至少 44×44px。

### 6.4 深度

优先使用背景亮度层级和 1px 半透明边框。仅浮层、菜单和拖拽对象使用阴影。工作区固定面板不使用大面积 blur。

## 7. 状态设计

所有异步功能至少包含：

| 状态 | 要求 |
|---|---|
| Empty | 告诉用户下一步，不做装饰性空状态插画 |
| Loading | 显示正在做什么；超过 2 秒显示步骤或预计等待 |
| Queued | 区分“等待”与“正在运行” |
| Running | 可停止；显示 Provider/模型和 elapsed |
| Needs review | 明确需要用户确认什么 |
| Success | 结果出现，同时保留来源和下一步 |
| Partial | 部分候选成功，允许保留成功项 |
| Error | SafeError、人话解释、重试/改配置动作 |
| Offline | 区分本地数据可用与远程生成不可用 |

## 8. 关键交互

### 8.1 对话生成

- Enter 发送，Shift+Enter 换行；
- 发送前若 Agent 计划会产生多次调用，先进入确认态；
- 参考素材显示缩略图与作用标签，不只显示文件名；
- 用户可点击“查看本轮上下文”。

### 8.2 选择候选

- 单击选中并检查；
- `F` 标记最终候选；
- `V` 创建变体；
- `E` 进入修改；
- 对比最多 4 个候选，保持相同缩放和背景。

### 8.3 修改

- 默认黑盒，先显示“保持/改变”；
- 执行前允许一键打开白盒；
- 每次修改创建子分支；
- 用户可以撤销工作区操作，但外部 API 调用不能“撤销计费”，文案必须区分。

### 8.4 Pipeline ↔ Canvas

- “展开为画布”使用空间连续动画，节点从步骤中展开；
- `prefers-reduced-motion` 下直接切换；
- 从画布切回时显示 freeform 内容数量；
- 不弹“转换成功”无意义 Toast，状态在视图标题中持续可见。

## 9. 无障碍

- 普通正文对比度 >= WCAG AA 4.5:1；
- 18px+ 大字 >= 3:1；
- 焦点环 2px 琥珀并有 2px offset；
- 所有 icon-only 按钮有 aria-label 和 tooltip；
- 状态不只依赖颜色；
- 画布关键操作提供键盘等价路径；
- 视频缩略图不自动播放声音；
- 动画尊重 `prefers-reduced-motion`；
- 可缩放界面不禁用浏览器/系统缩放；
- 生成内容可能闪烁时提供降低动态媒体选项。

## 10. 文案规范

### 使用
- “生成候选”而不是“创造奇迹”；
- “本轮使用了…”而不是“记住了所有内容”；
- “复现可信度：中”而不是“可完全复现”；
- “当前模型不支持 seed”而不是静默隐藏问题；
- “停止后已完成的调用仍可能计费”。

### 禁止
- 完美、绝对、100%、无限上下文；
- 神奇、革命性、重新定义；
- 不解释对象的“智能优化”；
- 把 Agent 内部步骤拟人化成无意义表演。

## 11. Slop 自审

| AI UI 常见问题 | 本规范约束 |
|---|---|
| 技术渐变 | UI chrome 不使用渐变 |
| 默认紫色 AI 色 | 使用克制暖琥珀，媒体承担色彩 |
| 三等分功能卡 | 工作区按任务层级而非营销卡片组织 |
| 玻璃拟态滥用 | 只允许浮层轻度透明，不用大面积 blur |
| 大数字填空间 | 不展示无行动意义的假指标 |
| 所有内容居中 | 左对齐、面板化、任务密度优先 |
| 错误表面 | 明确为 Operate + Command/Inspect |

目标 Slop Score：0–1 / 10。

## 12. 原型验收清单

- [ ] 项目模式选择可操作；
- [ ] Pipeline、画布、连接设置至少三屏可切换；
- [ ] 会话切换、Fork 和 Pin 状态可见；
- [ ] Agent 计划能展开、运行和停止；
- [ ] 黑/灰/白盒切换能改变检查器内容；
- [ ] 稳定锁可操作；
- [ ] 复现可信度有解释；
- [ ] 1280×720 无核心控件遮挡；
- [ ] 1440×900 主构图成立；
- [ ] 无控制台 JavaScript 错误；
- [ ] reduced-motion 有降级；
- [ ] 视觉自审不超过 1 分。
