# Gravuresse vNext UI/UX 规范

> 版本：v0.3（Simple First 对抗式审查后收敛）
> 日期：2026-07-16
> 规范源：根目录 `DESIGN.md`
> 原型：`docs/prototypes/gravuresse-vnext-prototype.html`
> 截图：`docs/prototypes/screens/`

---

## 1. 体验定位

### 1.1 主表面

Gravuresse 的默认产品表面不是营销页，也不是专家节点工具。v0.3 的界面组合为：

- **首页 / 连接页：Configure** —— 帮用户快速开始或完成连接，不先暴露技术模型；
- **创作工作区：Operate + Command/Inspect** —— 用户持续描述、生成、选择、修改；
- **详情 / 来源：Inspect** —— 选中对象后按需查看生成设置、来源记录和技术证据；
- **自由画布：Advanced Explore** —— 高级展开入口，不与默认步骤视图同级抢首屏。

### 1.2 Simple First 目标

```text
30 秒内知道怎么开始
3 分钟内完成一次：描述 → 生成 → 选择 → 修改
```

默认用户路径只呈现四件事：

1. 描述想法；
2. 生成候选；
3. 选择结果；
4. 简单修改。

底层 Project、WorkflowGraph、CanvasDocument、LineageGraph、CapabilityResolution、Recipe、Attempt 等对象可以完整存在，但默认 UI 不把这些词直接展示给用户。

### 1.3 设计目标

1. 让媒体和当前步骤成为视觉中心，界面退后；
2. 让复杂性按需展开，而不是在首屏堆概念；
3. 让“保持什么、改变什么”一眼可见；
4. 让用户知道助手正在做什么、能不能停止、下一步要确认什么；
5. 让连接设置回答“能不能生成图/视频/对话”，而不是让用户先理解 Provider 结构。

### 1.4 不做什么

- 不让新用户先选择 Pipeline / 自由画布；
- 不把黑盒 / 灰盒 / 白盒做成首屏三标签；
- 不常驻展示 Recipe、Lineage、Capability、Provider、ContextSnapshot、Attempt；
- 不使用紫蓝 AI 渐变、玻璃拟态、装饰性 Agent 头像；
- 不承诺“100%复现”“完整上下文”“绝对保持”。

## 2. 信息架构

```text
Gravuresse
├── 首页
│   ├── 开始创作（唯一主入口）
│   ├── 最近项目
│   └── 次级入口：导入项目 / 连接设置 / 高级自由画布
├── 创作工作区
│   ├── 创作记录（可折叠）
│   ├── 本轮参考（一行摘要）
│   ├── 步骤视图（默认主工作区）
│   │   ├── 描述想法
│   │   ├── 确认创作说明
│   │   ├── 生成候选
│   │   ├── 选择结果
│   │   ├── 简单修改
│   │   └── 导出 / 继续做视频
│   ├── 输入栏
│   ├── 详情检查器（默认折叠 / 选中后打开）
│   └── 高级：自由画布 / 来源记录 / 生成设置
└── 设置
    ├── 添加连接向导
    ├── 已连接能力：可对话 / 可生成图 / 可生成视频
    └── 技术详情：endpoint / 模型 revision / 请求模板 / 验证证据（折叠）
```

默认用户路径不得出现四等分顶栏“项目 / Pipeline / 无限画布 / 连接”。

## 3. 全局布局

### 3.1 Desktop Shell

```text
┌──────────── Title Bar / Project / Global Status ────────────┐
├─ Rail ─┬─ Creation Records ─┬─ Main Work Surface ─┬ Details ┤
│ 56px   │ 296–320px         │ flexible            │ 300–360 │
└────────┴───────────────────┴─────────────────────┴─────────┘
```

| 区域 | 默认 | 最小 | 行为 |
|---|---:|---:|---|
| 标题栏 | 40px | 40px | Electron drag region，控件禁用 drag |
| 图标项目栏 | 56px | 56px | 只放全局入口，不堆二级导航 |
| 创作记录栏 | 304px | 286px | 可收起；显示创作记录、本轮参考、助手阶段 |
| 主工作面 | 自适应 | 520px | 永远优先保留当前步骤与媒体候选 |
| 详情栏 | 332px | 300px | 默认折叠摘要；选中作品后显示详情 |
| 底部状态条 | 30px | 30px | 显示自动保存、任务、快捷键和门禁提醒 |

窗口宽度 `< 1180px` 时，只保留一个侧栏；`< 900px` 时进入专注模式，通过顶部按钮切换记录 / 详情。

### 3.2 主工作面优先级

1. 当前步骤问题；
2. 当前媒体 / 候选；
3. 主操作按钮；
4. 输入栏；
5. 详情与高级入口。

自由画布入口文案为：**高级：展开为自由画布**。它是从步骤视图展开，不是新建另一份项目。

## 4. 核心屏幕

### 4.1 首页 / Start Home

**目的**：30 秒内开始输入，不让用户先理解产品模式。

必须包含：

- 一个主入口：**开始创作**；
- 一个自然语言输入暗示；
- 最近项目；
- 次级入口：导入项目、连接设置、高级自由画布。

禁止：

- 首屏并列“Pipeline / 无限画布”模式卡；
- 首屏要求先配置 Provider；
- 用营销式大 Hero 替代真实开始入口。

截图：`01-start-home.png`

### 4.2 步骤式创作工作区

**目的**：把模糊想法推进到可比较候选。

步骤：

```text
描述想法 → 确认创作说明 → 生成候选 → 选择结果 → 简单修改 → 导出/继续视频
```

要求：

- 每一步只问一个问题；
- 当前步骤有唯一高强调动作；
- 未选择候选前不自动覆盖下游；
- 助手默认展示阶段、证据和停止按钮，不展示内部任务图；
- 输入栏上方或旁边显示一行“本轮参考”。

截图：`02-guided-creation.png`

### 4.3 简单修改

**目的**：用户无需理解黑盒 / 灰盒 / 白盒即可完成修改。

默认面板只显示：

```text
保持：人物、红伞、构图
改变：时间改清晨、雨改薄雾、灯光更自然
[生成修改版] [高级设置]
```

高级设置折叠项：

- 稳定锁；
- 生成设置差异；
- 来源记录；
- 可重做程度；
- 模型 / seed / 参数证据。

截图：`03-simple-edit.png`

### 4.4 连接向导

**目的**：按任务添加连接，而不是暴露技术控制台。

默认入口：

- 添加图像生成；
- 添加视频生成；
- 添加对话连接。

已连接列表显示：

- 可对话 / 可生成图 / 可生成视频；
- 认证状态；
- 最近验证；
- 是否能真实生成。

技术字段 endpoint、revision、请求模板、extraParams 必须折叠在技术详情中。密钥从不回显。

截图：`04-connection-wizard.png`

### 4.5 UI 规范板

**目的**：给设计、开发和后续 Agent 一张可视化约束图。

包含：

- 色彩 token；
- 字体层级；
- 桌面布局；
- 组件样例；
- Do / Don’t；
- Simple First 门禁。

截图：`05-ui-specification.png`

## 5. 核心组件

### 5.1 StartInput

- 高度：48px；
- 左侧为自然语言 placeholder；
- 右侧唯一主按钮“开始创作”；
- 不在此处要求选择模式。

### 5.2 ContextSummary

默认一行：

```text
本轮参考：创作说明 · 当前记录 · 2 张参考图
```

点击展开后显示来源、版本、Included / Excluded 和排除理由。默认不显示 ContextSnapshot、预算 token、裁剪算法等调试词。

### 5.3 AssistantStatus

显示：

- 当前阶段；
- 已完成阶段；
- 下一步；
- 停止按钮。

禁止：

- 展示内部思维链；
- 装饰性头像；
- 多 Agent 角色群聊；
- 伪造百分比进度。

### 5.4 GuidedStepCard

结构：

- 序号；
- 用户能理解的动作标签；
- 当前问题；
- 状态；
- 下一步动作。

状态必须同时有文字和视觉，不只依赖颜色。

### 5.5 CandidateCard

媒体优先。固定内容：

- 缩略图；
- 候选名；
- 差异说明；
- 选中标记；
- 下一步动作。

Prompt、模型、参数不叠在图上，进入详情查看。

### 5.6 SimpleEditPanel

默认只暴露：

- 保持；
- 改变；
- 生成修改版；
- 高级设置。

### 5.7 DetailsDrawer

选中作品后显示。默认折叠：

- 生成设置；
- 来源记录；
- 可重做程度；
- 稳定锁；
- 技术证据。

### 5.8 ConnectionWizard

按任务组织，不按 Provider 品牌组织。列表先显示能力和验证状态；技术字段折叠。

## 6. 视觉系统

规范 token 以根目录 `DESIGN.md` 为准。

### 6.1 色彩

- 近黑背景：`#0B0D10`；
- 面板：`#171B20`；
- 抬升面：`#1E232A`；
- 文本：`#F4F5F7 / #B6BDC7 / #7F8894`；
- 主交互：暖琥珀 `#F0A44B`；
- 成功、危险、信息仅用于语义状态。

琥珀只用于：

- 主操作；
- 当前步骤；
- 选中作品；
- 焦点；
- 需要确认的状态。

### 6.2 字体

```css
font-family: "Segoe UI Variable", "Microsoft YaHei UI", "PingFang SC", system-ui, sans-serif;
```

技术值：

```css
font-family: "Cascadia Mono", "SFMono-Regular", Consolas, monospace;
```

层级：

| 用途 | 尺寸 / 权重 |
|---|---|
| 项目标题 | 18–24px / 600–650 |
| 首页主标题 | 32–36px / 650，仅首页可用 |
| 面板标题 | 13–14px / 600 |
| 正文与控件 | 13–14px / 400–550 |
| 元数据 | 10–12px |

工作区不使用营销式超大展示字体。

### 6.3 空间与形状

- 4px 基础网格；
- 主节奏：8 / 12 / 16 / 24；
- 控件 radius：6px；
- 卡片 radius：8px；
- 弹层 radius：12px；
- 媒体卡 radius：6px；
- 标准控件高度：36px；
- 主输入高度：48px；
- 高频点击目标至少 36×36px，触摸场景至少 44×44px。

### 6.4 深度

优先使用背景亮度层级和 1px 边框。固定面板不使用玻璃模糊。只有菜单、对话框、拖拽对象使用克制阴影。

## 7. 状态设计

| 状态 | 要求 |
|---|---|
| Empty | 告诉用户下一步，不做装饰性插画 |
| Loading | 显示正在做什么；超过 2 秒显示阶段 |
| Queued | 区分等待和运行 |
| Running | 可停止；显示人话阶段 |
| Needs review | 明确需要用户确认什么 |
| Success | 结果出现，同时给出下一步 |
| Partial | 部分候选成功，允许保留成功项 |
| Error | SafeError、人话解释、重试/改配置动作 |
| Offline | 区分本地数据可用与远程生成不可用 |

## 8. 关键交互

### 8.1 开始创作

- 点击“开始创作”进入步骤视图；
- 若没有连接，不在首页阻塞，在第一次生成前提示；
- 可从最近项目恢复当前步骤。

### 8.2 选择候选

- 单击选中并打开详情；
- `F` 标记最终候选；
- `E` 进入简单修改；
- 对比最多 4 个候选，保持相同缩放和背景。

### 8.3 修改

- 默认先显示保持/改变；
- 每次修改创建子版本；
- 外部 API 调用不能“撤销计费”，文案必须区分撤销本地操作与取消远程任务。

### 8.4 展开为自由画布

- 从步骤视图点击“高级：展开为自由画布”；
- 步骤变成节点组；
- 当前选中路径高亮；
- 不可表示内容只读保留并说明原因；
- `prefers-reduced-motion` 下直接切换。

## 9. 无障碍

- 普通正文对比度 >= WCAG AA 4.5:1；
- 18px+ 大字 >= 3:1；
- 焦点环 2px 琥珀并有 2px offset；
- icon-only 按钮有 aria-label 和 tooltip；
- 状态不只依赖颜色；
- 关键画布操作提供键盘等价路径；
- 视频缩略图不自动播放声音；
- 动画尊重 `prefers-reduced-motion`；
- 不禁用系统缩放。

---

## 10. 文案规范

### 使用

- “开始创作”；
- “本轮参考”；
- “生成候选”；
- “保持 / 改变”；
- “高级设置”；
- “来源记录”；
- “可按当前设置再试一次，但结果可能有差异”；
- “当前连接不支持该参数”。

### 禁止

- 完美、绝对、100%、无限上下文；
- 神奇、革命性、重新定义；
- 智能优化但不解释对象；
- 多 Agent 角色表演；
- Pipeline、Recipe、Lineage、Capability、Provider、Attempt、ContextSnapshot 直接出现在默认用户路径。

## 11. Slop 自审

| AI UI 常见问题 | 本规范约束 |
|---|---|
| 技术渐变 | UI chrome 不使用渐变，媒体承担色彩 |
| 默认紫色 AI 色 | 单一暖琥珀作为交互信号 |
| 三等分功能卡 | 工作区按任务步骤组织，不按营销卡片组织 |
| 玻璃拟态滥用 | 固定面板不用 blur |
| 大数字填空间 | 不展示无行动意义的假指标 |
| 所有内容居中 | 工作区左对齐、面板化、任务密度优先 |
| 错误表面 | 明确 Configure / Operate / Inspect |

目标 Slop Score：0–1 / 10。

## 12. 原型验收清单

- [ ] 首页只有一个主入口“开始创作”；
- [ ] 新用户 30 秒内可进入想法输入；
- [ ] 步骤视图展示“描述 → 生成 → 选择 → 修改”核心路径；
- [ ] 1280×720 下核心路径无遮挡；
- [ ] 简单修改默认只显示“保持/改变”；
- [ ] 高级设置折叠但可展开；
- [ ] 助手运行可见、可停止，默认不显示内部任务图或角色群聊；
- [ ] 连接页默认显示能力和验证状态；
- [ ] endpoint / revision / 请求模板折叠到技术详情；
- [ ] Provider 不支持的参数不显示或明确禁用；
- [ ] 键盘焦点可见；
- [ ] reduced-motion 有降级；
- [ ] 无控制台 JavaScript 错误；
- [ ] 视觉自审不超过 1 分。
