# Security Policy / 安全策略

## Supported versions / 支持版本

Security fixes are provided for the latest published version. Please reproduce an issue on the latest release before reporting it.

安全修复面向最新发布版本。提交报告前，请先在最新版本上确认问题仍可复现。

## Private reporting / 私密报告

Do not disclose an unpatched vulnerability in a public issue, discussion, pull request, or log. Use GitHub's [private vulnerability reporting](https://github.com/sinepop/gravuresse/security/advisories/new).

请勿在公开 Issue、Discussion、Pull Request 或日志中披露未修复漏洞。请使用 GitHub 的[私密漏洞报告](https://github.com/sinepop/gravuresse/security/advisories/new)。

Include the affected version, impact, minimal reproduction steps, and any suggested mitigation. Redact credentials, tokens, personal data, and local filesystem paths. If a live credential was exposed, revoke or rotate it before sending the report; do not include the credential value.

报告应包含受影响版本、影响范围、最小复现步骤和可选修复建议。请删除凭据、令牌、个人数据和本地文件路径。若真实凭据已经暴露，请先撤销或轮换，且不要在报告中附上凭据值。

The project will acknowledge a complete report within seven days, validate the issue, and coordinate disclosure after a fix is available.

项目会在七天内确认完整报告，完成问题验证，并在修复可用后协调披露。
